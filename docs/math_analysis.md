# 数学分析：3D 黎曼度量场与当前超参设计

## 一、3D 中黎曼度量场：数学可行性分析

**结论先行**：框架在 3D 中数学上完全可行，且保留了 2D 的所有优雅性质。但从"可行"到"好用"，需要正视维数升高带来的三个本质挑战。

### 1. 核心数学结构在任意维数不变

基础对象都是维数无关的：

- **黎曼度量场** $g(x): \mathbb{R}^3 \to \mathrm{Sym}^+(3)$，在每一点赋予一个 $3 \times 3$ 正定对称矩阵。Cholesky 参数化同样适用（现在需要 6 个下三角分量：$l_{11}, l_{21}, l_{22}, l_{31}, l_{32}, l_{33}$），正定性由 $g = LL^\top + \epsilon I$ 保证。

- **原子的测地截断函数** $\phi_i(x) = \Phi(d_g(\mu_i, x) / r_i)$ 直接推广到 3D，距离仍用马氏近似：
  $$d_g \approx \sqrt{(x - \mu_i)^\top g(\mu_i) (x - \mu_i)}$$
  紧支撑性质保持不变。

- **体积渲染**从 2D 线积分自然变成 3D 体渲染（沿射线的密度积分），可微性不变。

- **凝聚损失（InfoNCE）**的正负样本定义基于测地距离：$d_g(\mu_i, \mu_j) < \tau$ 为正样本，$d_g(\mu_i, \mu_j) > \tau'$ 为负样本。这个定义在 3D 中同样有效。

### 2. 3D 独有的三个数学挑战

#### 挑战一：度量场学习的"过度自由度"

3D 度量在每个点有 **6 个自由度**（对称矩阵独立参数），而 2D 只有 3 个。如果使用体素网格，参数量是 $O(N^3 \times 6)$，极易过拟合或坍缩为恒等度量（$g = I$）。必须加强正则化：

- 度量平滑损失（$\|\nabla g\|_F^2$）和占位耦合损失在 3D 中的权重可能需要更高
- 否则度量场会退化为平庸解

#### 挑战二：测地邻接的稀疏性

- 在 2D 中，物体内的原子"测地球"容易重叠，正样本对较多
- 在 3D 中，体积增长立方，原子半径相对变小，正样本对数量可能骤减
- InfoNCE 需要足够的正样本才能有效工作

**缓解方案**：

- 增大原子初始半径，确保物体内部有足够重叠
- 适当放宽正样本阈值 $\tau$，甚至采用"软正样本"（连续权重而非硬阈值）

#### 挑战三：渲染与凝聚的梯度竞争加剧

- 3D 体积渲染的梯度信号比 2D 更稀疏（每条光线只能"看到"表面附近的一小段），导致原子在深度方向上约束较弱
- 凝聚损失如果过度依赖测地邻接（基于当前度量），可能把原子拉到错误的空间位置（因为度量本身还在学习中）
- 3D 中需要更强的位置正则化或渲染驱动的锚定，防止原子在深度轴上漂移

### 3. 可行性结论

数学上没有不可逾越的障碍。只要针对上述挑战调整训练策略（更强的度量正则、更大的初始半径、更深的位置监督），3D 系统可以复现甚至超越 2D 的聚类效果。目前 `train_3d.py` 的代码框架已经正确实现了 Cholesky 参数化的 3D 度量场和三线性插值，剩下的就是超参适配。

## 二、当前超参与损失设计的数学合理性分析

分析 `feat/clustering-breakthrough` 分支中采用的关键参数和策略。

### 1. InfoNCE 凝聚损失的数学形式与参数

**损失形式（简化）**：

$$\mathcal{L}_{\text{InfoNCE}} = -\frac{1}{|P|} \sum_{(i,j) \in P} \log \frac{\exp(\text{sim}(f_i, f_j) / T)}{\sum_{k \in N(i)} \exp(\text{sim}(f_i, f_k) / T)}$$

其中 $P$ 是基于测地距离的正样本对集合，$N(i)$ 是负样本集合。

#### 温度 $T$ 的作用

$T$ 控制特征相似度分布的锐度：

| $T$ | 行为 | 适用场景 |
|---|---|---|
| 小 ($T \ll 1$) | 关注难分负样本，梯度集中在特征空间边界 | 形成清晰簇（当前 $T=0.1$） |
| 大 ($T \approx 1$) | 趋近均匀分布，梯度更平滑 | 聚类不稳定时可放宽到 0.2~0.5 |

当前 $T=0.1$ 倾向于形成清晰的特征簇。如果聚类不稳定，可适当增大。

#### 正样本阈值 $\tau$

$\tau$ 定义了"测地距离多近算同一物体"。数学上，$\tau$ 应与原子平均半径 $2\bar{r}$ 相当，因为两个原子测地球重叠时，中心距约在 $r_i + r_j$ 左右。当前 $\tau=0.3$ 在 2D 中配合半径 0.12 是合理的（$0.3 \approx 2.5 \times \text{半径}$）。3D 中可能需要根据场景缩放调整。

#### 负样本选择的注意事项

InfoNCE 通常使用批次内所有其他样本作为负样本。但包含"假负样本"（同物体但测地距离因度量场学习不完全而偏大）会误导特征学习，相当于噪声标签，降低聚类纯度。

**缓解方案**：使用测地距离大于 $\tau'$（如 $\tau' = 2\tau$）的硬阈值筛选负样本，而不是全部纳入。

### 2. 占位耦合权重 ($w_{\text{vol}} = 0.2$) 的合理性

占位耦合损失 $\mathcal{E}_{\text{vol}}$ 强制物体内部度量迹小，背景迹大。权重分析：

- 渲染损失 L1 在 **0.05~0.1** 量级
- $\mathcal{E}_{\text{vol}}$ 原始值在 **1~5** 量级
- 权重 0.2 意味着贡献约 **0.2~1.0**，与渲染损失相近量级 ✅

2D 实验结果显示 trace 分离明显（物体内 0.5~2，背景 5~9），证实了这个权重的有效性。3D 中可保持相近比例。

### 3. 位置正则化权重 ($w_{\text{pos}} = 5.0$) 的数学分析

位置正则化 $\mathcal{E}_{\text{pos}} = \frac{1}{N} \sum_i d(\mu_i, \text{mask})$ 惩罚原子离开物体区域，与渲染损失形成"拔河"：

- 渲染损失试图把原子拉到颜色匹配的背景（灰色）
- 位置正则化往回拽
- 权重 5.0 在 2D 中找到平衡点（覆盖率 82%）

**注意**：这个权重过高会抑制原子的探索能力，可能导致原子卡在物体边缘。未来可以设计衰减策略：在训练后期降低 $w_{\text{pos}}$，让原子在物体内部微调。

### 4. KMeans 初始化时机的选择

在 Phase 1 结束（epoch 250）时用原子位置做 KMeans (K=2) 初始化特征向量，本质上提供了**空间引导的特征先验**：

- ✅ 打破了对称性，让 InfoNCE 一开始就有正样本梯度
- ✅ 当前简单形状下效果很好（ARI 0.175）
- ⚠️ 如果 KMeans 本身错误划分原子，偏置会误导聚类
- 🔜 更普适做法：使用谱聚类（基于测地距离矩阵）而非欧氏 KMeans

### 5. 总损失的数值稳定性

当前总损失组成：
$$\mathcal{E}_{\text{total}} = \mathcal{E}_{\text{render}} + \lambda_{\text{met}} \mathcal{E}_{\text{met}} + \lambda_{\text{vol}} \mathcal{E}_{\text{vol}} + \lambda_{\text{pos}} \mathcal{E}_{\text{pos}} + \lambda_{\text{coh}} \mathcal{E}_{\text{coh}}$$

- $\mathcal{E}_{\text{coh}}$ 采用 InfoNCE，天然有界于 $[0, \log N]$，不会再出现之前 -97 的发散问题
- 总损失稳定在 **+4.9**，证明数值平衡已初步达成 ✅

## 三、从数学出发的下一步优化建议

### 1. 3D 度量场初始化

不要让度量场从随机开始，而是预训练一个"各向同性"度量（$g = \alpha I$，$\alpha$ 可学习），先用渲染损失 + 占位耦合损失训练几百步，等物体/背景迹初步分离后再引入原子。这可以避免早期随机度量导致原子误聚类。

### 2. 3D 中正样本的"软化"

定义正样本权重：

$$w_{ij} = \exp\left(-\frac{d_g(\mu_i, \mu_j)^2}{2\sigma^2}\right)$$

其中 $\sigma$ 与原子半径相关。这样远处的原子贡献较小但仍有梯度，缓解硬阈值导致的稀疏问题。

### 3. 特征分离的辅助项

即使 InfoNCE 已经鼓励簇内紧致和簇间分离，仍可加入一个极弱的特征方差保持项（如 0.01 权重），作为安全网防止全部特征坍缩到单点——这在 3D 中因正样本稀疏更容易发生。

### 4. 参数自适应

$\tau$ 和 $T$ 不要固定，可以根据当前度量场的迹分布动态调整。例如：

$$\tau = 2 \times \text{median}(r_i) \times \sqrt{\text{trace}(g_{\text{avg}})}$$

让正样本定义随度量场成长而演变。

## 总结

当前的数学框架在 3D 中依然坚实，参数选择经过 2D 的迭代验证，符合损失函数的量级平衡和梯度动力学。需要警惕的是维数升高带来的**稀疏性**和**度量自由度**问题，但这些问题都可以通过增强正则化和调整样本策略来解决，不存在根本性的数学障碍。
