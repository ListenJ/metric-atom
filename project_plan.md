# MetricAtom 项目计划

## 1. 项目愿景
构建一个基于黎曼度量和紧支撑感知原子的三维场景理解系统，从多视角图像（及未来扩展的WiFi CSI）中自动涌现物体实例，无需任何预训练语义模型或外部2D分割引导。

## 2. 核心假设
- 在黎曼度量场 g(x) 上定义的有界感知原子，通过联合优化外观重建和度量结构，能自然凝聚为与真实物体边界对齐的簇。
- 度量的各向异性可以直接编码物体内部（高密度、小特征值）与空空间（低密度、大特征值）的差异，从而驱动无监督聚类。

## 3. 技术路线概览

```
多视角图像 / WiFi CSI
        │
        ▼
黎曼度量场 g(x) ←── 联合优化
        │
感知原子（紧支撑、特征向量、存在概率）
        │
可微体积渲染 ─── 重建损失
        │
凝聚损失 ─── 自发聚类
        │
物体实例（簇）
```

## 4. 系统架构

```
src/
├── geometry/      # 黎曼度量场、Cholesky参数化、测地线近似
├── atoms/         # 原子定义、集合管理、空间索引
├── rendering/     # 光线采样、体积积分、CUDA核（后期）
├── losses/        # 重建、度量平滑、占位耦合、凝聚
├── data/          # 合成数据生成、数据加载
├── training/      # 训练循环、验证、早停
└── visualization/ # 度量场、原子分布、聚类可视化
```

## 5. 开发阶段

### Phase 0: 项目脚手架（第1天）
- 创建目录结构，配置依赖和约束文件。
- 搭建 OpenCode 多代理工作流。

### Phase 1: 2D原型验证（第1-2周）
- 实现2D度量场、原子、渲染器、损失函数。
- 生成简单几何形状的合成多视角数据。
- 训练并验证：原子在无分割标签下是否按形状成簇。
- **交付物**：可运行的 Jupyter Notebook 与 Python 脚本，可视化报告。

### Phase 2: 3D静态场景扩展（第3-6周）
- 将2D模块升级至3D（度量场3D体素/MLP，3D原子，3D体积渲染）。
- 使用CARLA或Blender合成数据集。
- 引入稀疏深度（雷达模拟）作为初始化和锚点。
- 评估聚类质量（ARI, NMI）与渲染质量（PSNR/SSIM）。
- **交付物**：3D核心代码库，论文草稿方法论部分。

### Phase 3: 动态与多模态扩展（第7-12周）
- 引入时间维度 → 度量场 g(x,t)。
- 集成WiFi CSI数据（使用MM-Fi等公开数据集），实现跨模态一致性能量项 E_csi。
- 在车内/室内场景验证遮挡物体持续性。
- **交付物**：完整系统原型，提交arXiv论文。

## 6. 任务拆分与约束

已分解为9个独立任务（Task 0 ~ Task 8），每个任务附有详细约束词，用于防止AI生成时引入无关依赖。

任务间依赖关系：
- **可并行**：Task1（度量场）、Task2（原子）、Task4（渲染器）、Task5（损失函数）。
- **依赖**：Task3（原子集合）依赖Task2；Task6（数据生成）独立；Task7（训练）依赖1-5；Task8（可视化）依赖全部。

## 7. 开发环境与约束

- **语言**：Python 3.10+
- **框架**：PyTorch 2.x，不引入任何高级训练框架（初期）
- **视觉库**：OpenCV, matplotlib
- **计算**：纯PyTorch体积渲染（2D），后期可加CUDA kernel
- **禁忌**：禁止使用COLMAP、预先训练的SAM/CLIP、任何高斯泼溅库

## 8. 协作模式（OpenCode）

- 主代理负责调度，审查PR，运行测试。
- 多个子代理独立开发模块，通过约束文件确保代码风格与架构一致性。
- 代码合并前强制通过对应的 pytest 单元测试。

## 9. 成功标准

- **2D原型**：两个形状的合成场景中，原子自动分为两簇，且簇边界与形状掩码IoU > 0.85。
- **3D静态**：多物体室内合成场景，无监督分割的NMI > 0.8，渲染SSIM > 0.9。
- **多模态**：在视觉遮挡区域，CSI的引入能维持原子存在性 > 90% 的真实物体。

## 10. 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| 度量场退化为恒等矩阵 | 强占位耦合损失 + 背景度量锚定 |
| 特征坍缩（所有原子同特征） | 凝聚损失中的惩罚项 + 特征熵奖励 |
| 训练时间过长 | 先在2D验证，稀疏空间索引，MoE优化 |

## 11. 时间线（总计12周）

- 第1周：脚手架 + 2D模块开发
- 第2周：2D集成训练 + 可视化验证
- 第3-6周：3D重构 + 合成数据训练
- 第7-9周：时间扩展 + 多模态CSI集成
- 第10-12周：实景测试 + 论文撰写
