# MetricAtom 项目计划

## 1. 项目愿景

构建一个基于黎曼度量和紧支撑感知原子的三维场景理解系统，从多视角图像中自动涌现物体实例，无需任何预训练语义模型或外部2D分割引导。

## 2. 核心假设

- 在黎曼度量场 g(x) 上定义的有界感知原子，通过联合优化外观重建和度量结构，能自然凝聚为与真实物体边界对齐的簇。
- 度量的各向异性可以直接编码物体内部（高密度、小特征值）与空空间（低密度、大特征值）的差异，从而驱动无监督聚类。

## 3. 技术路线概览

```
多视角图像
     │
     ▼
黎曼度量场 g(x) ←── 联合优化
     │
感知原子（紧支撑、特征向量、存在概率）
     │
可微体积渲染 ─── 重建损失
     │
凝聚损失 ───── 自发聚类
     │
物体实例（簇）
```

## 4. 系统架构

```
src/
├── geometry/      # 黎曼度量场、Cholesky参数化、测地线近似
├── atoms/         # 原子定义、集合管理、空间索引
├── rendering/     # 光线采样、体积积分（向量化）
├── losses/        # 重建、度量平滑、占位耦合、凝聚
├── data/          # 合成数据生成、数据加载
├── training/      # 训练循环、剪枝、播种
└── visualization/ # 度量场、原子分布、聚类评估
```

### 已验证的核心假设

- ✅ **渲染收敛**：L1 loss 从 0.5+ 下降到 0.04-0.08
- ✅ **度量场分离**：物体内部 trace < 1，背景 trace > 9
- ✅ **特征多样性保持**：新方差排斥使 feat_std 稳定在 0.3-0.5（对比旧版坍缩到 0.005）
- ✅ **动态原子管理**：播种+剪枝使物体内原子比例从 5% 提升到 15-22%
- ⬜ **物体对齐聚类**：ARI/NMI 未达标（负值），特征正确但空间覆盖率不足

### 未验证的瓶颈

| 瓶颈 | 现状 | 根因 |
|------|------|------|
| 空间覆盖率 | 物体仅占图像 4-5%，~10/90 原子在物体上 | 播种机制优先高误差区域，但误差分布广 |
| 特征-空间耦合 | 特征已多样化但未形成空间聚类 | 吸引力不足以克服随机特征分布 |
| ARI/NMI | -0.1 ~ 0.0（负值） | 有效样本太少，无法分离为2簇 |

## 7. 下一步实验计划

### 实验 1：提高空间覆盖率（建议优先级最高）

目标：物体内原子从 15-22% 提升到 40%+

| 方案 | 方法 | 预期效果 |
|------|------|----------|
| 1a | 大半径初始化 (r=0.2-0.3) | 初始覆盖面积增大 4x |
| 1b | 播种直接落在物体掩码上（使用合成数据的真实掩码） | 100%种子命中物体 |
| 1c | 多尺度原子：全局粗覆盖 + 局部细节 | 同时保覆盖和细节 |
| 1d | 播种时检查目标像素颜色，避免播在背景 | 种子质量提升 |

### 实验 2：改进特征-空间耦合

前提：覆盖率达标后。

- 在吸引项中增大 spatial 权重（当前 C 归一化可能过弱）
- 或引入显式空间引导：在 Phase 2 前，用 KMeans(mu) 初始化特征
- 或使用对比学习形式：正样本对 = 测地邻接原子，负样本对 = 远距离原子

### 实验 3：替换合成数据为真实图像

- 用 MNIST / 简单室内场景照片替代合成形状
- 验证真实数据上度量场是否仍具备分离能力

### 实验 4：3D 扩展

- 2D 验证通过后，扩至 3D（度量场 MLP、3D 原子、3D 渲染）
- 使用 COLMAP 输出作为弱初始化

## 9. 开发环境与约束

- **语言**：Python 3.10+
- **框架**：PyTorch 2.11，Intel MKL
- **视觉库**：OpenCV, matplotlib
- **计算**：纯 PyTorch 体积渲染（向量化，CPU）
- **禁忌**：禁止使用 COLMAP、预训练SAM/CLIP、高斯泼溅库
- **加速**：MKL + OpenMP（多线程），Intel oneAPI 就绪

## 10. 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| 度量场退化为恒等矩阵 | 占位耦合损失 + 背景度量锚定 |
| 特征坍缩 | 方差最大化排斥（已解 ✅） |
| 原子覆盖率不足 | 播种+剪枝（持续改进中） |
| CPU 训练时间过长 | MKL 多线程 + 向量化渲染（已解 ✅） |
| GPU 驱动不可用 | MKL CPU 回退（已解 ✅） |
